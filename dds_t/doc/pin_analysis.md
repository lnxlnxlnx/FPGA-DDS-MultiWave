# 引脚分配与项目功能分析

## 1 项目概述

| 项目 | 内容 |
|------|------|
| **顶层实体** | `okok4_test` (BDF 原理图) |
| **目标器件** | Cyclone IV E — **EP4CE6E22C8** |

## 4 时序与约束

### 4.1 时钟域

| 时钟信号 | 频率 | 来源 | 驱动对象 |
|---------|------|------|---------|
| `clk` | 12 MHz | 外部晶振 PIN_23 | 所有模块 |
| `ad_clk` | 12 MHz / ~100 kHz | 直通或内部分频 | ADC 采样时钟 |
| `da_clk` | 12 MHz / ~100 kHz | 同 `ad_clk` | DAC 转换时钟 |

### 4.2 关键分频参数

| 用途 | 计数值 | 分频比 | 输出频率/周期 |
|------|--------|--------|-------------|
| ADC 采样时钟 | 60 (半周期) | 120 | ~100 kHz |
| 按键消抖 | 100,000 | 100,000 | 8.3 ms |
| 数码管位扫描 | 6,000 | 6,000 | ~0.5 ms/位 |
| 频率计闸门 | 11,999,999 | 12,000,000 | 1 s |

### 4.3 IO 约束

- **IO 标准**: `STRATIX_DEVICE_IO_STANDARD = "2.5 V"`
- **ADC 数据**: 8 bit 并行输入，与 `ad_clk` 同步采集
- **DAC 数据**: 12 bit 并行输出，在 `da_clk` 上升沿锁存
- **数码管**: 共阳，段码低电平点亮；位选高电平选中

### 4.4 复位

- `rst_n` 为 **低有效异步复位**
- 复位时：状态机回到 `IDLE`，DA 输出强制清零，频率计计数清零

## 5 数据路径

### 5.1 采集回放路径

```
ADC ──► ad_data[7:0]
         │
         ├── 直通模式: ──► {ad_data, 4'b0} ──► da_data[11:0] ──► DAC
         │
         └── 录制模式 (mode=0, start↓):
               FSM → 写 RAM[addr] ← sample_tick (100kHz)
               4096 采样点 → 约 41ms 采集窗口
               │
         └── 回放模式 (mode=1, start↓):
               FSM → 读 RAM[addr] → 输出到 da_data
               单次: 播完停；循环: 播完重播
```

### 5.2 频率测量路径

```
ad_data[7:0]
    │
    ▼
迟滞过零检测 (hysteresis: 5/250)
    │
    ▼
1s 闸门计数 → freq_x10 (单位: Hz×10)
    │
    ▼
seg_driver:
  32b bin → BCD → 7段译码 → 扫描输出
  显示格式: XXXX.X Hz (个位小数点恒定点亮)
```

## 7 .qsf 关键约束

```tcl
# 器件与顶层
set_global_assignment -name FAMILY "Cyclone IV E"
set_global_assignment -name DEVICE EP4CE6E22C8
set_global_assignment -name TOP_LEVEL_ENTITY okok4_test

# IO 电平
set_global_assignment -name STRATIX_DEVICE_IO_STANDARD "2.5 V"

# 引脚分配 (详见第 2 节)
set_location_assignment PIN_23 -to clk
set_location_assignment PIN_51 -to rst_n
# ... 49 个引脚分配
```

> 注意: 当前 `okok4_test.qsf` 中**未定义**任何 `.sdc` 时序约束文件 (未设置 `SDC_FILE`)。实际编译时 Quartus 会对 12 MHz 主时钟自动推导约束。如需精确的建立/保持时间分析，建议补充 `.sdc` 文件。
